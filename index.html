Introduction

Sass est un préprocesseur qui permet de compiler et/ou interprété du Sass ou Scss en CSS.

Un préprocesseur permet de transformer un type de données en un autre type de données.

Outils

  • Node JS
  • Sass

Pour installer Sass de manière globale sur votre machine :

npm install -g sass

Pour lancer la compilation :

sass --watch dossier_sass/fichier.scss dossier_css/fichier.css

sass --watch dossier_sass:dossier_css

Syntax

Lorsque nous utilisons Sass, nous avons la posibilité entre deux formes d'écriture : .sass ou .scss

La syntaxe Scss est une surcouche du CSS, ce qui signifie que toutes les instructions valides en CSS le sont également en Scss. Cette syntaxe est plus facile à utiliser grâce à sa ressemblance avec le CSS.

La seconde syntaxe est celle d'origine de Sass. Elle s'écrit de manière indentée et sans accolade et point virgule.

Exemple de code en Sass

@mixin button-base() {
		@include typography(button);
		@include ripple-surface;
		@include ripple-radius-bounded;
	
		display: inline-flex;
		position: relative;
		height: $button-height;
		border: none;
		vertical-align: middle;
	
		&:hover { cursor: pointer; }
	
		&:disabled {
				color: $mdc-button-disabled-ink-color;
				cursor: default;
				pointer-events: none;
		}
}

Exemple de code en Scss

@mixin button-base()
		@include typography(button)
		@include ripple-surface
		@include ripple-radius-bounded

		display: inline-flex
		position: relative
		height: $button-height
		border: none
		vertical-align: middle

		&:hover
				cursor: pointer

		&:disabled
				color: $mdc-button-disabled-ink-color
				cursor: default
				pointer-events: none

Nesting

Nesting signifie Imbrication en anglais.

Lorsque vous écrivez du code HTML, vous avez probablement remarqué qu'il possède une hiérarchie imbriquée et visuelle claire. CSS, en revanche, ne le fait pas.

Sass permet d'imbriquer les sélecteurs CSS de la même manière que vous le faite pour HTML.

Attention tout de fois que trop imbriquer rendra plus difficile la maintenance du code. C'est généralement considéré comme une mauvaise pratique

Exemple de Nesting

nav {
	ul {
		margin: 0;
		padding: 0;
		list-style: none;
	}

	li { display: inline-block; }

	a {
		display: block;
		padding: 6px 12px;
		text-decoration: none;
	}
}

Sélecteur parent : &

a {
	text-decoration: none;

	&:hover {
		text-decoration: underline wavy;
	}

}

Exemple Scss

#demo {
	&-child {
		margin: .5em;
	}

	&child {
		margin: .5em;
	}

	&.child {
		margin: .5em;
	}

	& .child {
		margin: .5em;
	}
}

Exemple CSS


#demo-child {
	margin: .5em;
}

#demochild {
	margin: .5em;
}

#demo.child {
	margin: .5em;
}

#demo .child {
	margin: .5em;
}

Le sélecteur parent permet de reprendre le nom du parent. Le nom du parent sera injecté dans le code après compilation.

Nested properties

L'utilisation de propriétés imbriquées permet de ne pas répéter plusieurs fois ces propriétés. Nous pouvons écrire comme namespace (espace de noms), le nom de la propriété. Par exemple, toutes les propriétés font (font-style, font-size, font-weight, etc.) peuvent être raccourcies et donc n'écrire qu'une seule fois l'espace de noms.

Exemple SCSS

.enlarge {
	font-size: 14px;
	transition: {
		property: font-size;
		duration: 4s;
		delay: 2s;
	}

	&:hover { font-size: 36px; }
}

Exemple SCSS

.enlarge {
	font-size: 14px;
	transition-property: font-size;
	transition-duration: 4s;
	transition-delay: 2s;
}
.enlarge:hover {
	font-size: 36px;
}

Ecriture raccourcie Scss

.info-page {
	margin: auto {
		bottom: 10px;
		top: 2px;
	}
}

Ecriture raccourcie CSS

.info-page {
	margin: auto;
	margin-bottom: 10px;
	margin-top: 2px;
}

Variables

La notion de variable en Sass est la même que dans n'importe quel langage de programmation. Le but étant de stocker une valeur sous un alias et d'utiliser l'alias plutôt que la valeur elle-même à divers endroit. Cela permet ainsi d'éviter de recopier la valeur plusieurs fois mais aussi de ne modifier la valeur qu'à un seul endroit pour toutes les modifier.

Une variable en Sass s'écrit de la manière suivante :

$variable_name : some value;

Ainsi une variable commencera par un dollar et finira par un point virgule. La structure se base à la déclaration d'une propriété CSS classique.

Scss

$txtcolor:#008000;
$fontSize: 20px;

p{
	color:$txtcolor;
		font-size:$fontSize;
}

CSS

p {
	color: #008000;
	font-size: 20px;
}

La portée des variables ! Lorsqu'une variable est déclarée dans un bloc, elle n'est accessible uniquement dans ce bloc.

Vous pouvez ajouter un !global à la fin d'une déclaration de variable pour passer la variable globale et plus locale

Interpolation

L'interpolation est fortement utilisé dans Sass. Elle permet d'intégrer un résultat dans une expression.

Il suffit juste d'écrire une expression entre moustache précédé d'un diese.

#{ expression }

Scss

@mixin corner-icon($name, $top-or-bottom, $left-or-right) {
	.icon-#{$name} {
		background-image: url("/icons/#{$name}.svg");
		position: absolute;
		#{$top-or-bottom}: 0;
		#{$left-or-right}: 0;
	}
}

@include corner-icon("mail", top, left);

CSS

.icon-mail {
	background-image: url("/icons/mail.svg");
	position: absolute;
	top: 0;
	left: 0;
}

Data types

Le type de données est le type d'une information qui nécessite de déclarer le type de données pour chaque objet de données.

Il existe plusieurs types de données tels que :

Numbers

Les nombres en Sass ont deux composants : la valeur et l'unité (optionnel)

Exemple : 12 100px 5px * 2px 5.2e3

Lors des opérations, vous ne pouvez pas calculer 2 unités différentes. Pour régler le soucis, vous pouvez donner une unité à la première valeur et omettre la suivante.

Strings

Les chaines de caractères sont des séquences de caractères.

Sass supporte deux types de chaînes de caractères : les chaînes entre guillemets, comme "Helvetica Neue", et les chaînes sans guillements (également appelées identificateurs)

Exemple : ".widget:hover" bold

Vous pouvez utiliser la fonction string.unquote(".widget:hover"); // .widget:hover ou la fonction string.quote(bold); // "bold" pour récupérer une chaine de caractères entre guillemets ou pas.

Pour échapper un caractère, vous pouvez utiliser le caractère d'échappement : \

Colors

Sass intègre un support pour les valeurs des couleurs. Il est possible de créer des couleurs de plusieurs façons : 

@debug #f2ece4; // #f2ece4
@debug #b37399aa; // rgba(179, 115, 153, 67%)
@debug midnightblue; // #191970
@debug rgb(204, 102, 153); // #c69
@debug rgba(107, 113, 127, 0.8); // rgba(107, 113, 127, 0.8)
@debug hsl(228, 7%, 86%); // #dadbdf
@debug hsla(20, 20%, 85%, 0.7); // rgb(225, 215, 210, 0.7)

Lists

Les listes continennent une séquence d'autres valeurs. Dans Sass, les éléments des listes peuvent être séparés par des virgules (Helvetica, Arial, sans-serif), des espaces (10px 15px 0 0) ou des barres obliques tant qu'ils sont cohérents dans la liste.

Contrairement aux autres langages, les listes dans Sass ne nécessitent pas de crochets spéciaux ; toutes les expressions séparées par des espaces ou des virgules comptent comme une liste.

Exemple : margin: 10px auto 15px; font-family: Arial, Helvetica, sans-serif;

Maps

Les maps sont des listes qui contiennent des paires de clefs et de valeurs. Pour récupérer une valeur, il faudra entrer la clef associée. Dans le même principe qu'un dictionnaire avec des mots et leur définition.

Les maps sont stockées dans une variable et construites entre parenthèse. Chaque paires est construite en stipulant expression: valeur et séparées par une virgule.

Comme chaque valeur est définie par une clef, celle-ci doit être unique au sein de la map. Il ne peut donc pas avoir deux clefs identiques.

Exemple : $font-weights: ("regular": 400, "medium": 500, "bold": 700); $colors: (1: #89C3AB, 2: #636363);

Pour récupérer un élément de la liste, il vous faudra utiliser la méthode map-get($variable, clef)

Booleans

Les bolléens sont des valeurs logiques vrai et faux. En plus de leurs formes littérales, les booléens sont renvoyés par des opérateurs d'égalité et relationnels, ainsi que par de nombreuses fonctions intégrées telles que math.comparable() ou map.has-key()

Scss

@mixin avatar($size, $circle: false) {
	width: $size;
	height: $size;

	@if $circle {
		border-radius: $size / 2;
	}
}

.square-av {
	@include avatar(100px, $circle: false);
}
.circle-av {
	@include avatar(100px, $circle: true);
}

CSS

.square-av {
	width: 100px;
	height: 100px;
}

.circle-av {
	width: 100px;
	height: 100px;
	border-radius: 50px;
}

Operations

Sass prend en charge une poignée d'opérateurs utiles pour travailler avec différentes valeurs.

Opérateurs arithmétique : + - * /

Les opérateurs mathématiques sont utilisés par réaliser des opérations classiques entre les nombres, seulement pour les nombres avec la même unité (px => px, vh => vh, etc...)

Les deux membres de part et d'autre du signe sont calculés sur base de l'unité à gauche du signe.

Attention lors d'une multiplication, l'unité y est également multiplié, il faut donc l'omettre pour la partie de droite.

12px + 12px // 24px 1in - 10px; // 0.8958333333in 12px * 2px // 24px*px 12px / 12px // 12px / 12px

Pour la division, vous devez mettre l'expression entre () pour la réaliser.

Opérateurs d'égalité : == !=

Les opérateurs d'égalité permettent de vérifier l'égalité entre le membre de gauche et celui de droite du signe.

== vérifie si les deux valeurs sont les mêmes. Dans le cas où ils sont les mêmes, la valeur true est renvoyée, dans le cas contraire false.

!= fait l'inverse du ==.

Opérateurs relationnels : < > <= >=

L'opérateur relationnels déterminent si les nombres sont plus grands ou plus petits les uns que les autres. Ils convertissent automatiquement entre les unités compatibles.

> retourne si le premier élément est supérieur par rapport au second.

< retourne si le premier élément est inférieur par rapport au second.

>= retourne si le premier élément est supérieur OU égal par rapport au second.

<= retourne si le premier élément est inférieur OU égal par rapport au second.

Opérateurs de concaténation : + -

La concaténation permet de coller deux valeurs ensemble. (cfr. Interpolation).

+ retourne une chaine de caractères contenant les deux expressions. Si l'expression de gauche est entre guillemet, le résultat le sera également. Sans guillemet dans le cas contraire.

- retourne une chaine de caractères sans guillemet contenant les deux expressions séparées par -.

Il s'agit d'un ancien opérateur, utilisez l'interpolation à la place.

Opérateurs booléens : not and or

Contrairement aux autres langages, Sass utilise des mots plutôt que des symbales pour les opérateurs booléens.

not retourne la valeur opposée de l'expression. Elle transforme vrai en faux et faux en vrai.

and retourne vrai si la valeur des deux expressions est à vrai. Sinon faux.

or retourne vrai si l'une des valeurs des deux expression est à vrai. Sinon faux.

@import directive

Sass étend la règle @import de CSS avec la possibilité d'importer des feuilles de style Sass et Css.

Ainsi nous pouvons importer des mixins, fonctions, variables et plusieurs feuilles de styles ensemble.

L'import se fait dès la compilation contrairement aux imports css qui réalisent des requêtes HTTP.

la directive @import va devenir dépréciée. Il faudra priorisé la directive @use à la place.

Plus d'informations au lien suivant : https://sass-lang.com/documentation/at-rules/import

SCSS

// foundation/_code.scss
code {
	padding: .25em;
	line-height: 0;
}
							
// foundation/_lists.scss
ul, ol {
  text-align: left;

  & & {
    padding: {
      bottom: 0;
      left: 0;
    }
  }
}
							
// style.scss
@import 'foundation/code', 'foundation/lists';

CSS

code {
	padding: .25em;
	line-height: 0;
}

ul, ol {
	text-align: left;
}
ul ul, ol ol {
	padding-bottom: 0;
	padding-left: 0;
}

Ce ne serait pas amusant d'écrire des URL absolues pour chaque feuille de style que vous importez, donc l'algorithme de Sass pour trouver un fichier à importer le rend un peu plus facile.

Pour commencer, vous n'avez pas besoin d'écrire explicitement l'extension du fichier que vous souhaitez importer

Exemple : @import "variables" va automatiquement charger variables.scss, variables.sass, ou variables.css

Partials

Les fichiers partiels sont des fichiers destinés à être importés uniquement. Pour indiquer au compilateur que ce sont des fichiers partiels, nous indiqueront un "_" devant les noms des fichiers.

Exemple : _variables.scss

Lorsque le compilateur compile tous les fichiers, les fichiers partiels sont ignorés et ne sont qu'importés dans les fichiers avec un @import à l'intérieur.

Pour voir un exemple, rendez-vous dans l'exemple du précédent chapitre. (cfr. @import directive)

Une utilisation intéressante avec les imports est le nesting.

Vous pouvez importer un fichier CSS pour un sélecteur particulier :

SCSS

// _theme.scss
pre, code {
	font-family: 'Source Code Pro', Helvetica, Arial;
	border-radius: 4px;
}
						
// style.scss
.theme-sample {
	@import "theme";
}

CSS

.theme-sample pre, .theme-sample code {
	font-family: 'Source Code Pro', Helvetica, Arial;
	border-radius: 4px;
}
